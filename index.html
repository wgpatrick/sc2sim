The Optimizer/Value-frontier/Opponent sections below are Protoss-only for now.

⚡ Optimizer — fastest army at the enemy (Protoss only)

        

          Pick a target army. The template optimizer exhaustively searches ~31,600 build variants
          (worker count, production buildings, gas, proxy vs. home) within a fixed build SHAPE -- the true
          optimum within that template. The raw-sequence GA searches Action[] sequences directly (no
          fixed shape), seeded from the template's own answer so it can never do worse, and in practice beats
          it by 10-18s on every example in this project by discovering orderings the template can't express.
        

        

        

          

            
            
          

          
          
        

        

        

      


      

        
📈 Value frontier — best army value by time T (Protoss only)

        

          Sweeps unit COUNTS (not just one fixed target) and reports the Pareto frontier of
          unitValue() — sqrt(dps × (hp + shields)) — delivered at the enemy by time T. This is a
          ranking heuristic, not a combat resolver: it ignores upgrades, range, splash, and unit counters entirely.

🥊 Opponent — safety against a REAL recorded game (Protoss only)

        

          Replays a real Terran/Zerg opponent's recorded build order through their own race's simulator to get a
          cumulative fighting-VALUE-over-time curve straight from a game someone actually played, then compares it
          against the build order currently loaded in the simulator above (top-left panel — edit it and re-compare).
          A build that's fast but leaves you with less value than a real opponent had at that moment is a build
          that loses to that opener, regardless of how it ranks in isolation. These 3 sample games are trimmed to
          their opening 5 minutes and embedded directly (this is a UI convenience subset —
          npm run opponent draws on the full 20-replay corpus).
