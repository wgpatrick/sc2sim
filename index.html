⚡ Search for a specific army

Pick exactly the army you want, and compare two search strategies against each other: exact search tries every combination inside one fixed build shape (~thousands of variants, guaranteed best within that shape); smart search evolves build sequences directly with no fixed shape, and in practice beats the exact search by 10-20s on almost every target by finding orderings the shape can't express.

📈 Value frontier — the biggest army you can have, by time T

Given only the unit types you check below, this finds — for every point in time — the most raw fighting power (value) any searched build can deliver to the enemy by then. It answers "what's the strongest possible army I could have at 6 minutes," not "what's the best composition" — it freely mixes unit types and doesn't know about countering, splash, or range. It also doesn't model upgrades — an upgraded Zealot ball is much stronger than the same count of stock Zealots, and that difference is entirely invisible here today (see How this works).

🥊 Opponent — safety against a REAL recorded game

        

          Takes a real opponent's actual recorded build order from a ladder replay and runs it through the
          same simulator, producing a value-over-time curve for what they really did — then compares it
          against the build currently loaded in the panel above. A build that's fast but leaves you
          with less value than a real opponent had at that moment is a build that loses to that opener,
          regardless of how it ranks in isolation. These 3 sample games are trimmed to their opening 5
          minutes and embedded directly (a UI convenience subset — npm run opponent draws on
          the full 26-replay corpus). How this comparison works →
